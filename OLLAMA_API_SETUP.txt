OLLAMA API SETUP GUIDE

═══════════════════════════════════════════════════════════════════════════════

TWO WAYS TO USE OLLAMA WITH YOUR RAG SYSTEM:

1️⃣ LOCAL OLLAMA (No API Key Needed)
═════════════════════════════════════

This is the simplest option if you have Ollama running locally.

Setup:
  • Download Ollama: https://ollama.ai
  • Run: ollama run mistral
  • In app sidebar:
    - Toggle "Use Ollama API" = ON
    - API Key: leave empty
    - API URL: http://localhost:11434/api/chat
  • Done!

When to use: Local machine, development, testing


2️⃣ REMOTE OLLAMA API (With API Key)
═══════════════════════════════════

If you have a remote Ollama server (hosted on cloud/VM).

Setup:
  • Get your Ollama server URL: https://your-server.com
  • Get your API key from the server
  • In app sidebar:
    - Toggle "Use Ollama API" = ON
    - API Key: paste your key here (or set OLLAMA_API_KEY env var)
    - API URL: https://your-server.com/api/chat
  • Done!

When to use: Cloud deployment, shared server, production


═══════════════════════════════════════════════════════════════════════════════

ENVIRONMENT VARIABLES (To avoid re-entering in UI):

PowerShell (Windows):
  $env:OLLAMA_API_KEY = "your_api_key_here"
  $env:OLLAMA_API_URL = "http://localhost:11434/api/chat"

Bash (Mac/Linux):
  export OLLAMA_API_KEY="your_api_key_here"
  export OLLAMA_API_URL="http://localhost:11434/api/chat"

.env file (all systems):
  OLLAMA_API_KEY=your_api_key_here
  OLLAMA_API_URL=http://localhost:11434/api/chat


═══════════════════════════════════════════════════════════════════════════════

ERROR SOLUTIONS:

❌ "401 Unauthorized"
   → Your API key is missing or invalid
   → Check OLLAMA_API_KEY is set correctly
   → Verify the API endpoint is correct

❌ "404 Not Found"
   → The API URL is wrong
   → Check the endpoint format: http://localhost:11434/api/chat
   → For remote: https://your-server.com/api/chat

❌ "Connection refused"
   → Ollama server is not running
   → Terminal 1: ollama run mistral
   → Wait 10 seconds for server to start
   → Then try again

❌ "Timeout"
   → Server is slow or overloaded
   → Increase timeout in code if needed
   → Check server status

═══════════════════════════════════════════════════════════════════════════════

CORRECT API ENDPOINT FORMAT:

✅ Local (Ollama on same machine):
   http://localhost:11434/api/chat

✅ Remote (Ollama on another machine):
   http://your-ip-or-domain:11434/api/chat
   https://your-ip-or-domain:11434/api/chat

✅ Cloud hosted Ollama:
   https://your-service.com/api/chat
   https://your-service.com/v1/chat/completions

If unsure, check your Ollama server logs or documentation.

RECOMMENDED SETUP:

For best experience:

1. Start with LOCAL mode (no API key needed):
   • Install Ollama locally
   • Run: ollama run mistral
   • Use API URL: http://localhost:11434/api/chat
   • No API key required

2. Once working, you can switch to remote if needed:
   • Get a remote Ollama server
   • Use its API URL and key
   • Toggle in app

═══════════════════════════════════════════════════════════════════════════════

QUICK TEST (Command Line):

PowerShell:
  $env:OLLAMA_API_URL = "http://localhost:11434/api/chat"
  curl -X POST $env:OLLAMA_API_URL `
    -H "Content-Type: application/json" `
    -d '{"model":"mistral","prompt":"Hello","stream":false}'

Bash:
  curl -X POST http://localhost:11434/api/chat \
    -H "Content-Type: application/json" \
    -d '{"model":"mistral","prompt":"Hello","stream":false}'

Should return a JSON response with the generated text.

═══════════════════════════════════════════════════════════════════════════════

STILL NOT WORKING?

1. Check Ollama is running:
   curl http://localhost:11434/api/tags
   (should return list of models)

2. Check API key (if remote):
   Make sure OLLAMA_API_KEY matches what server expects

3. Check firewall:
   Remote access might be blocked by firewall
   Open port 11434 or the API port

4. Check server logs:
   Look at Ollama server logs for error details

5. Try local mode first:
   Get it working locally, then move to remote
